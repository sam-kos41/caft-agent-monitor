Four ways to measure an AI agent — and why each reduces to the action sequence

In the dataset, this exact record — frozen-sample row , instance — is labelled succeeded: its stored target outcome is resolved, i.e. the agent's final patch passed that repository's test suite (the RESOLVED badge above is that same dataset label). The four constructs vary all the way through it regardless of that outcome — which is exactly why the interesting question is discriminant: does any of them add anything beyond the action sequence itself? On 2,000 trajectories, none do. The table below the trajectory is the finding; the trajectory shows what each lens actually measures, step by step.

What “IT” means here. IT = information-theoretic structure of the action sequence: the statistical shape of what the agent did and in what order — how predictable each action is given the ones before it (mutual information), how repetitive / compressible the sequence is (compression ratio), how far its action mix drifts (KL divergence), and how varied it is (entropy). It is computed from the sequence of action types alone — no reasoning text, no outcome. Why so much reduces to it: workload, perception, error-recovery and thought–action coherence are each, ultimately, patterns in that same action sequence. Once its information-theoretic structure is in the model, each added construct carries little the sequence had not already encoded — so each predicts the outcome alone yet adds ≈0 beyond IT.

Each construct predicts task outcome on its own. Not one adds predictive value beyond the information-theoretic action sequence (locked decision rule: does it beat baseline + IT?). The agent-native lens is the sharpest case — least correlated with IT (0.43) yet still adds essentially zero.
